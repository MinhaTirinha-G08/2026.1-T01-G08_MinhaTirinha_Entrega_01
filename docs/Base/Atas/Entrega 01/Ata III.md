## 1. Objetivo

## 2. Discussões

### 2.1 Levantamento de requisitos
[cite_start]O grupo trabalhou na identificação e definição dos requisitos para consolidar as principais funcionalidades da aplicação. Os requisitos funcionais resumidos incluem:
* [cite_start]Criação de tirinhas.
* [cite_start]Edição de elementos visuais como personagens, cenários e textos.
* [cite_start]Salvamento e gerenciamento das tirinhas criadas.
* [cite_start]Compartilhamento das tirinhas.

### 2.2 Requisitos não funcionais
[cite_start]Foram definidos critérios de qualidade para a aplicação:
* [cite_start]Usabilidade e facilidade de uso.
* [cite_start]Interface intuitiva.
* [cite_start]Desempenho adequado.
* [cite_start]Compatibilidade com diferentes dispositivos.

### 2.3 Refinamento das ideias
[cite_start]Os requisitos foram discutidos de forma colaborativa para alinhar o entendimento entre os membros e garantir consistência com o escopo.

### 2.4 Relação com etapas anteriores
[cite_start]O grupo reconheceu que a definição dos requisitos está diretamente ligada às atividades anteriores, como o brainstorming e a definição do público-alvo.
